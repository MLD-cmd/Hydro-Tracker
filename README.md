# HydroTracker 🌴💧

A friendly, island-themed hydration tracker built with Flutter and powered by
Supabase. Create an account, log every sip, keep a streak going, earn badges,
and let your daily goal adapt to your body and the weather — with an AI coach
that cheers you on.

Meet **Splash**, your hydration buddy, who reacts to your progress and gently
nudges you back to water when the less-hydrating drinks start to pile up.

## Features

- **Accounts & sync** — sign up / sign in with Supabase Auth. Your entries,
  goal, custom drinks, and settings live in Postgres (guarded by Row-Level
  Security) and follow you across devices.
- **AI hydration coach** — the splash screen greets you with a short, fresh tip
  generated by a Supabase Edge Function (Groq LLM). The model key stays
  server-side; the app only sends your progress.
- **Quick logging** — log Water, Coconut, Juice, Tea, Coffee — or your own
  custom drinks — in any amount, via a stepper, real-world bottle/cup presets,
  or a typed value.
- **Custom drinks & presets** — add your own drink types (name, hydration
  weight, icon, color) and quick-log amount presets; they appear everywhere.
- **Hydration weighting** — each drink counts toward your goal by how much it
  actually hydrates (water 1.0, coconut 1.1, juice 0.85, tea 0.9, coffee 0.6).
- **Personalized goal** — get a recommended daily goal from your weight and
  activity level (~35 ml/kg), or set your own manually — your choice always wins.
- **Smart Goal with live weather** — fetches the live temperature for a chosen
  Philippine city (via Open-Meteo) and raises your goal on hot days, with an
  offline estimate as fallback.
- **Today's Mix** — per-drink bars showing how much of each drink you've had
  today and its share of the day's mix (custom drinks included).
- **Hydration Buddy** — an animated mascot whose mood tracks your progress and
  who nudges toward water when coffee/juice out-pour it.
- **Stats — "Hydration Tides"** — Today / Week / Month views with a goal line,
  tappable day details, a timeline of when you drank, streaks, a personal best,
  achievements, and a rotating hydration tip.
- **Celebrations** — a confetti moment when you hit your goal or unlock a badge.
- **Reminders** — scheduled local hydration reminders with a custom start/end
  window, interval, and Quiet Hours; notifications carry one-tap quick-log
  buttons.
- **History** — a full, day-grouped log of every drink, with edit and
  (soft) delete.
- **Personalization** — editable profile (name + avatar, stored in Supabase
  Storage), three environment themes, and a one-time onboarding intro.

> **Offline-friendly:** entries are cached on-device (`shared_preferences`) so
> logging works without a connection; writes reconcile to Supabase on the next
> launch.

## Tech stack

- **Flutter** (Material 3)
- **Supabase** — Auth, Postgres + Row-Level Security, Storage (avatars), and an
  Edge Function for the AI coach
- **Groq** — the LLM behind the coach (called only from the Edge Function)
- **shared_preferences** — local cache + settings
- **http** — live weather from the free [Open-Meteo](https://open-meteo.com) API
- **flutter_local_notifications** + **timezone** — scheduled reminders
- **image_picker** + **path_provider** — avatar selection

## Getting started

```bash
flutter pub get
flutter run
```

The app ships with a default Supabase URL + **anon (publishable)** key, so it
runs out of the box — the anon key is safe in client code because RLS guards the
data. To point at your own project:

```bash
flutter run \
  --dart-define=SUPABASE_URL=https://YOUR-PROJECT.supabase.co \
  --dart-define=SUPABASE_ANON_KEY=YOUR_ANON_KEY
```

### Backend (Supabase)

The `supabase/` directory holds the schema and the coach function:

- `migrations/` — tables (`profiles`, `water_entries`, `drink_types`,
  `bottle_presets`), RLS policies, and the avatars storage bucket.
- `functions/coach/` — the AI coach Edge Function.

The coach's model key is **never** in the app. Set it as a function secret and
deploy:

```bash
supabase secrets set GROQ_API_KEY=...        # required
supabase functions deploy coach              # JWT-verified: signed-in users only
```

## Project structure

```
lib/
├── main.dart            # entry point → splash screen
├── config/              # Supabase connection (URL + anon key via --dart-define)
├── models/              # settings, drink types, entries, achievements, etc.
├── domain/              # pure logic: goal calc, streaks, forecast, catalogs
├── repositories/        # entries, drink types, presets, profile, settings
├── services/            # auth, weather, notifications, AI coach
├── screens/             # splash, onboarding, sign in/up, dashboard, stats,
│                        # settings, history
├── state/               # environment theme
├── theme/               # colors + text styles
├── utils/               # helpers (e.g. password strength)
└── widgets/             # buddy, water circle, celebration overlay, cards, etc.

supabase/
├── migrations/          # schema + RLS
└── functions/coach/     # AI coach Edge Function
```
